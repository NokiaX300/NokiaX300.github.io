十二、常见问题

Q：npm install 报 UNC 路径 / cmd.exe 错误？

WSL 内调用了 Windows 版 npm。检查 which npm 是否指向 /mnt/c/...；若是，确认第二节的 PATH 过滤已生效，执行 hash -r 或重开终端。

Q：swift --version 报 KeyError: '--version'？

ms-swift 4.x CLI 没有全局 --version 参数，需用 python -c "import swift; print(swift.__version__)" 或 pip show ms-swift 查看版本；验证 CLI 可用可执行 swift sft --help。

Q：pip install flash-attn 编译失败？

确认已安装 cuda-toolkit 且 nvcc --version 正常；或暂时跳过，LoRA 微调不强制依赖 flash-attn。

Q：ModelScope 下载速度慢？

配置 MODELSCOPE_CACHE 到本地大盘；优先使用 Python API 下载。

Q：显存不足（OOM）？

减小 per_device_train_batch_size、启用 --gradient_checkpointing true、使用 LoRA 而非全参微调；A6000 48 GB 可支撑 7B 级模型 LoRA，70B 需量化或多卡。

Q：WSL2 下训练性能不如原生 Linux？

WSL2 GPU 直通性能接近原生，但磁盘 I/O 跨文件系统（如项目位于 /mnt/c/）会显著变慢。务必将代码与数据放在 Linux 文件系统（如 ~/）下。

Q：torch.cuda.is_available() 为 True，但 torch.empty(..., device='cuda') 报 CUDA driver error: unknown error？

检查是否设置了 PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True。该选项在 WSL2 下与 CUDA 驱动不兼容，会导致显存分配失败。执行 unset PYTORCH_CUDA_ALLOC_CONF 或从 ~/.bashrc 中注释掉该行，然后 source ~/.bashrc 再试。

Q：conda activate 后 Python 版本不对？

检查 ~/.bashrc 中是否有 export PATH="/usr/bin:..." 之类覆盖了 Conda 路径的写法，应改用第二节的 /mnt/ 过滤方案。

小结

  • 本文环境基于 Ubuntu 26.04 WSL2 + RTX A6000 + CUDA 13.2 驱动，微调框架为 ms-swift 4.x + Python 3.12。
  • 推荐路径：系统更新 → Miniconda（清华镜像加速）→ PyTorch (cu128) → ModelScope → ms-swift，再用小数据集验证训练流程。
  • WSL 开发注意 Linux 与 Windows 工具链隔离：过滤 /mnt/ 路径，不要把 /usr/bin 硬插到 PATH 最前。
  • 基座模型与数据集缓存放在 /home/nokia/root/models；训练输出与日志放在 /home/nokia/root/projects/train。
  • CUDA Toolkit 在 Ubuntu 26.04 上为 13.x；纯微调场景通常不需要单独安装，PyTorch cu128 wheel 即可。
