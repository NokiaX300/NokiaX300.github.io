LaTeX 快速入门

本文不定时更新，记录我在学习使用 LaTeX 来 记笔记、写论文 时遇到的一些困难与思考。
希望能给刚接触 LaTeX 的同学一条「能尽快跑通、自己生成一份完整 PDF」的路线。
遇到问题时，我优先查阅的是《一份（不太）简短的 LATEX 2ε 介绍》，这里也强烈推荐。
文中很多做法只是我的个人选择，不一定是唯一方案。

TeX、LaTeX、TeX Live、TeXworks 的区别

结论：下载 TeX Live（本体）和 VS Code（写作用的编辑器）即可。

  • TeX：最底层的排版语言。
  • LaTeX：对 TeX 的一层封装，让写文档更方便。
  • TeX Live：把 TeX、LaTeX 以及常用宏包、工具都打包在一起的「大合集」，一般直接装它。
  • TeXworks / 其他编辑器：写 .tex 源码的工具，可以用记事本、TeXworks，也可以像我一样用 VS Code。

我个人推荐：TeX Live + VS Code，界面友好、插件丰富。

VS Code 作为 LaTeX 编辑器

这部分在前面那篇 TexLive 2021 安装指南 里也有详细说明，这里只列关键步骤：

  1. 去官网下载安装 Visual Studio Code。
  2. 打开 VS Code，左边「扩展」中搜索并安装 LaTeX Workshop。
  3. 如果习惯中文界面，可以再安装扩展 Chinese (Simplified)。
  4. 按照教程把推荐的配置粘贴到 settings.json 中并保存，方便一键编译 & 预览。

日常如何新建 / 打开 LaTeX 文件

我个人的习惯：

  1. 在 VS Code 里点击 「文件 → 打开文件夹」，选一个用来放 LaTeX 工程的空目录。
  2. 在资源管理器里 新建文件，命名时以 .tex 结尾，例如 notes.tex。

之后光标就会跳到编辑区域，可以开始写代码。写完后，点击右上角绿色小三角（或用 LaTeX Workshop 的快捷键）进行编译。

  • 左下角会显示「building」等状态，完成后变成对勾，出错会显示叉号。
  • 目录里会自动生成一堆辅助文件，其中 .pdf 就是最终文档。
  • 通常可以用 Ctrl + Alt + V 在 VS Code 右侧打开 PDF 预览（具体快捷键以插件设置为准）。

以后要打开旧文档，只要在 VS Code 里点「文件 → 打开文件」，选择你的 .tex 文件即可。

🍉 文档类（Document Class）

每个 LaTeX 源文件开头都要用 \documentclass{...} 指定文档类型，
比如 article、report、book、beamer 等。

🍉 引擎（Engine）

负责把 源代码 编译成 最终文档 的「后端程序」，比如 pdfLaTeX、XeLaTeX、LuaLaTeX 等。
中文排版一般推荐使用 XeLaTeX。

各种后缀的文件都是什么？

第一次编译时，目录里会出现一堆新文件，大致含义如下（更详细的说明见手册第 7 页）：

后缀 含义说明
.tex 源代码文件
.pdf 编译生成的最终文档
.aux 主辅助文件，记录交叉引用、目录、参考文献等信息
.log 编译时生成的日志，排错时非常有用
.cls 文档类文件，文档类名称与文件名通常是一一对应的

⚡ 关于空格和行距

常用的一些空格与换行方式：

  • \\：换行，不缩进。可以加可选参数控制行距，例如 \\[15pt]。
  • \par：换段，有缩进。
  • \noindent：取消当前段落的首行缩进。
